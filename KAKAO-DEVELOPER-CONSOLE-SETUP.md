# 카카오 개발자 콘솔 설정 가이드

## ⚠️ 현재 발생 중인 에러 해결

콘솔에 다음과 같은 에러가 발생하고 있습니다:
```
카카오맵 API 오류: 403
{errorType: 'NotAuthorizedError', message: 'App(Ohun) disabled OPEN_MAP_AND_LOCAL service.'}
```

이 에러는 **로컬 서비스가 비활성화**되어 있어서 발생합니다.

## 🔧 해결 방법

### 1단계: 카카오 개발자 콘솔 접속
- [카카오 개발자 콘솔](https://developers.kakao.com/) 접속
- 애플리케이션 "Ohun" 선택

### 2단계: 로컬 서비스 활성화

1. **"제품 설정" 탭 클릭**
2. **"로컬" 찾기**
3. **"로컬" 활성화 버튼 클릭** (또는 "활성화 설정" 클릭)
4. **저장**

**참고**: 
- 로컬 서비스는 주소 검색, 좌표 변환에 사용됩니다
- 무료 제공량: 일 10만 건

### 3단계: 플랫폼 설정 확인

1. **"플랫폼" 탭 클릭**
2. **Web 플랫폼**에 다음 도메인 등록:
   - 개발: `http://localhost:5173`
   - 운영: 실제 도메인 (예: `https://ohun.com`)

### 4단계: 앱 키 확인

1. **"앱 키" 탭 클릭**
2. 다음 키들이 모두 있는지 확인:
   - ✅ **REST API 키** (주소 변환용)
   - ✅ **JavaScript 키** (지도 표시용)

## ✅ 설정 완료 후

1. **클라이언트 재시작** (필수!)
   ```bash
   cd client
   npm run dev
   ```

2. **브라우저에서 테스트**
   - "새 모임 만들기" → 주소 입력
   - 콘솔에 403 에러가 사라져야 함
   - 마커가 정상적으로 이동해야 함

## 📝 참고

- **로컬 서비스**: 주소 → 좌표 변환 (지오코딩), 좌표 → 주소 변환 (역지오코딩)
- **JavaScript 키**: 지도 표시
- 두 가지 모두 필요합니다!

## 🐛 여전히 문제가 있는 경우

### 스크립트 로드 실패
- JavaScript 키가 올바른지 확인
- `.env` 파일에 `VITE_KAKAO_JAVASCRIPT_KEY` 설정 확인
- 클라이언트 재시작 확인

### 403 에러 계속 발생
- 로컬 서비스가 활성화되었는지 확인
- REST API 키가 올바른지 확인
- 플랫폼 설정에 `http://localhost:5173` 등록 확인

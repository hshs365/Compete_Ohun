# JavaScript SDK 도메인 최종 설정 가이드

## ✅ 올바른 페이지를 찾으셨습니다!

현재 보고 계신 페이지가 **JavaScript SDK 도메인 설정** 페이지입니다.

## 📝 설정 방법

### 1단계: JavaScript SDK 도메인 입력

1. **"JavaScript SDK 도메인" 섹션 찾기**
   - 페이지에 "JavaScript SDK 도메인" 라벨이 보입니다
   - 경고 메시지: "도메인 정보를 등록하지 않으면 JavaScript 키를 사용할 수 없습니다"

2. **입력 필드에 도메인 입력**
   - Placeholder: "(예: https://example.com)"
   - **여기에 입력**: `http://localhost:5173`
   - ⚠️ **주의**: `https://`가 아닌 `http://`를 사용하세요 (로컬 개발 환경)

3. **'+' 버튼 클릭**
   - 입력 필드 오른쪽의 '+' 버튼을 클릭하여 도메인 추가

4. **저장**
   - 페이지 하단의 "저장" 또는 "확인" 버튼 클릭

## ✅ 설정 완료 후

### 1단계: 브라우저 완전히 닫기
- 모든 브라우저 창 닫기

### 2단계: 클라이언트 재시작
```bash
cd client
npm run dev
```

### 3단계: 브라우저 다시 열기
- `http://localhost:5173` 접속

### 4단계: 테스트
1. **F12 → Network 탭**
2. 필터에 "kakao" 입력
3. `dapi.kakao.com/v2/maps/sdk.js` 요청 확인
4. **상태 코드가 200 OK**인지 확인

## 📋 전체 설정 체크리스트

### 필수 설정 (순서대로):

1. **추가 기능 신청** ✅
   - [ ] "앱 설정" > "앱" > "추가 기능 신청"에서 카카오맵 신청
   - [ ] 승인 대기 및 승인 완료 확인

2. **카카오맵 활성화** ✅
   - [ ] "제품 설정" > "카카오맵"에서 상태를 ON으로 변경

3. **JavaScript SDK 도메인 등록** ✅ (현재 단계)
   - [ ] "플랫폼 키" > "JavaScript 키" > "JavaScript SDK 도메인"
   - [ ] `http://localhost:5173` 입력 및 저장

4. **로컬 API 활성화** (선택사항)
   - [ ] "제품 설정"에서 "로컬" 활성화 (주소 변환 기능 사용 시)

## ⚠️ 주의사항

- **도메인 형식**: `http://localhost:5173` (프로토콜 포함)
- **저장 필수**: 입력 후 반드시 저장 버튼 클릭
- **변경 사항 반영 시간**: 1-2분 소요될 수 있음

## 🎯 다음 단계

JavaScript SDK 도메인을 등록한 후:
1. 브라우저 완전히 닫기
2. 클라이언트 재시작
3. 브라우저 다시 열기
4. 카카오맵이 정상 작동하는지 확인

## 🆘 문제 해결

### 여전히 스크립트 로드 실패가 발생하는 경우:
1. **도메인이 정확히 입력되었는지 확인**
   - `http://localhost:5173` (정확히 이 형식)
2. **저장 버튼을 클릭했는지 확인**
3. **1-2분 대기 후 다시 시도**
4. **브라우저 캐시 클리어** (Ctrl+Shift+R)

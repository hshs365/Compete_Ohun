# 카카오맵 설정 최종 체크리스트

## ✅ 현재 상태 확인

### JavaScript SDK 도메인 설정 ✅
- **도메인**: `http://localhost:5173` ✅ (올바르게 입력됨)
- **저장 필요**: 하단의 "저장" 버튼 클릭

## 📋 전체 설정 체크리스트

### 1. 추가 기능 신청 ✅
- [ ] "앱 설정" > "앱" > "추가 기능 신청"에서 카카오맵 신청 완료
- [ ] 승인 완료 확인

**확인 방법:**
- "앱 설정" > "앱" > "추가 기능 신청" 페이지에서
- 카카오맵의 상태가 "승인됨" 또는 "활성화됨"인지 확인

### 2. 카카오맵 활성화 ✅
- [ ] "제품 설정" > "카카오맵"에서 상태를 ON으로 변경

**확인 방법:**
- "제품 설정" > "카카오맵" 페이지에서
- "사용 설정" > "상태" 토글이 ON인지 확인

### 3. JavaScript SDK 도메인 등록 ✅ (현재 단계)
- [x] "플랫폼 키" > "JavaScript 키" > "JavaScript SDK 도메인"
- [x] `http://localhost:5173` 입력됨 ✅
- [ ] **저장 버튼 클릭** (아직 안 누르셨다면)

### 4. 로컬 API 활성화 (선택사항, 주소 변환 기능 사용 시)
- [ ] "제품 설정"에서 "로컬" 활성화

**확인 방법:**
- "제품 설정" 메뉴에서 "로컬" 찾기
- 활성화되어 있는지 확인

## 🎯 지금 해야 할 일

### 1단계: 현재 페이지에서 저장
1. **하단의 "저장" 버튼 클릭**
2. **저장 완료 메시지 확인**

### 2단계: 나머지 설정 확인

#### 추가 기능 신청 확인:
1. "앱 설정" > "앱" > "추가 기능 신청" 페이지로 이동
2. 카카오맵이 승인되었는지 확인
3. 승인되지 않았다면 승인 대기

#### 카카오맵 활성화 확인:
1. "제품 설정" > "카카오맵" 페이지로 이동
2. "사용 설정" > "상태" 토글이 ON인지 확인
3. OFF라면 ON으로 변경

#### 로컬 API 활성화 (선택사항):
1. "제품 설정"에서 "로컬" 찾기
2. 활성화되어 있지 않다면 활성화 (주소 변환 기능 사용 시)

## ✅ 저장 후 테스트

### 1단계: 브라우저 완전히 닫기
- 모든 브라우저 창 닫기

### 2단계: 클라이언트 재시작
```bash
cd client
npm run dev
```

### 3단계: 브라우저 다시 열기
- `http://localhost:5173` 접속

### 4단계: 테스트
1. **F12 → Network 탭**
2. 필터에 "kakao" 입력
3. `dapi.kakao.com/v2/maps/sdk.js` 요청 확인
4. **상태 코드가 200 OK**인지 확인

5. **콘솔 확인**:
   - "카카오맵 스크립트 로드 시도" 메시지 확인
   - "카카오맵 스크립트 로드 완료" 메시지 확인
   - 에러가 없어야 함

6. **지도 표시 확인**:
   - 홈화면에 카카오맵이 표시되는지 확인
   - "새 모임 만들기" → "지도 보기"에서 카카오맵이 표시되는지 확인

## ⚠️ 주의사항

- **저장 필수**: JavaScript SDK 도메인 입력 후 반드시 "저장" 버튼 클릭
- **변경 사항 반영 시간**: 1-2분 소요될 수 있음
- **브라우저 재시작**: 설정 변경 후 브라우저를 완전히 닫고 다시 열어야 함

## 🆘 문제 해결

### 저장 후에도 스크립트 로드 실패가 발생하는 경우:

1. **추가 기능 신청 확인**:
   - 카카오맵이 승인되었는지 확인
   - 승인되지 않았다면 승인 대기

2. **카카오맵 활성화 확인**:
   - "제품 설정" > "카카오맵"에서 상태가 ON인지 확인

3. **도메인 형식 확인**:
   - `http://localhost:5173` (정확히 이 형식)
   - `https://`가 아닌 `http://` 사용

4. **1-2분 대기 후 다시 시도**

# 사업자번호 선점 악용 방지 방안

## 문제

사업자번호는 **한 계정에만 등록**되도록 검증되어 있습니다.  
악의적인 사용자가 타인의 사업자번호를 먼저 등록하면, 실제 사업자는 해당 번호로 가입·변경이 불가능해질 수 있습니다.

## 현재 구현

- **회원가입**: 사업자번호 API 검증 후, 이미 다른 계정에서 사용 중이면 `"이미 등록된 사업자번호입니다"` 로 가입 불가.
- **사업자번호 변경**: 비밀번호 확인 → API 검증 → 다른 계정에서 사용 중이면 변경 불가.

## 권장 완화·방지 방안

1. **사업자 가입 시 전화번호 인증 강화**  
   사업자 회원가입 시 전화번호 인증을 필수로 두어, 무작위 다수 계정 생성을 어렵게 합니다.

2. **분쟁 처리 절차(고객센터)**  
   "이 사업자번호는 제 것이며, 먼저 등록한 계정이 악용이다"라고 주장하는 경우:
   - 사업자등록증 등 서류로 본인·사업자 확인
   - 확인 후: 기존 등록 계정의 사업자번호 해제 또는 해당 계정 정지, 신청 계정에 사업자번호 등록

3. **동일 IP/기기 다수 사업자 가입 제한**  
   동일 IP 또는 기기에서 짧은 시간에 여러 사업자 계정이 가입되는 경우 의심 트래픽으로 보고, 추가 인증 또는 가입 제한을 검토합니다.

4. **추가 인증 단계(선택)**  
   사업자번호 등록 시 이메일/문자 인증 코드, 또는 공동인증서/간편인증 등 2단계 인증을 도입하면 선점 악용을 더 줄일 수 있습니다.

위 방안은 운영 정책과 개발 일정에 맞춰 단계적으로 적용하는 것을 권장합니다.

# 실명인증 구현 옵션

## 현재 상태
- ✅ 전화번호 인증 (SMS 인증번호) - 이미 구현됨
- ✅ 생년월일 필드 - 이미 존재함

## 실명인증 구현 방법

### 1. 기본 방법 (이름 저장)
**설명**: 이름 필드를 추가하고 전화번호 인증과 함께 저장
- **장점**: 구현이 간단함
- **단점**: 실제 실명인증이 아님 (이름만 저장)
- **적용 시기**: 개발/테스트 단계

### 2. 아이핀(i-PIN) 인증
**설명**: 한국정보인증(KICA)의 아이핀 API 사용
- **장점**: 공식 실명인증, 주민등록번호 대체
- **단점**: API 연동 필요, 유료 서비스
- **비용**: 건당 약 100~200원
- **API 문서**: https://www.kica.or.kr/

### 3. PASS 인증
**설명**: 통신사 앱(삼성패스, KB모바일, 신한패스 등)을 통한 인증
- **장점**: 사용자 편의성 높음, 무료
- **단점**: 통신사별 API 연동 필요, 복잡함
- **비용**: 무료 (통신사별 상이)

### 4. 본인인증 대행 서비스
**설명**: 나이스평가정보, KG모빌리언스 등 대행 서비스 사용
- **장점**: 여러 인증 방법 통합 제공
- **단점**: 유료, 서비스 의존성
- **비용**: 건당 약 100~300원
- **주요 서비스**:
  - 나이스평가정보: https://www.niceid.co.kr/
  - KG모빌리언스: https://www.kgmobilians.com/

### 5. 간단한 조합 인증 (권장: 초기 단계)
**설명**: 이름 + 전화번호 + 생년월일 조합으로 기본 검증
- **장점**: 구현 간단, 추가 비용 없음
- **단점**: 실제 실명인증은 아님
- **적용 시기**: MVP 단계, 나중에 외부 API로 업그레이드 가능

## 권장 구현 순서

### Phase 1: 기본 구조 (현재)
1. ✅ 전화번호 인증 완료
2. ✅ 생년월일 필드 존재

### Phase 2: 이름 필드 추가 (즉시 가능)
1. User 엔티티에 `realName` 필드 추가
2. 회원가입 시 이름 입력 받기
3. 전화번호 인증과 함께 저장

### Phase 3: 외부 API 연동 (운영 단계)
1. 본인인증 대행 서비스 선택 (나이스평가정보 등)
2. API 연동
3. 실명인증 완료 플래그 추가

## 구현 예시 (Phase 2)

```typescript
// User Entity
@Column({ type: 'varchar', length: 50, nullable: true })
realName: string | null; // 실명

@Column({ type: 'boolean', default: false })
realNameVerified: boolean; // 실명인증 완료 여부
```

## 참고사항
- 실제 실명인증은 외부 API가 필요합니다
- 전화번호 인증만으로는 실명인증이 아닙니다
- 운영 환경에서는 반드시 공식 인증 서비스를 사용해야 합니다

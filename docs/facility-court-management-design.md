# 시설·구장 계층 구조 및 등록 편의성 기획

## 1. 배경 및 목적

하나의 건물·단지 내에 **여러 층** 또는 **여러 면**(1구장, 2구장 등)이 존재하는 경우:

- **시설 관리자**: 공간을 명확히 구분하여 등록할 수 있어야 함  
- **유저**: 자신이 뛸 정확한 위치를 직관적으로 파악해야 함

---

## 2. 데이터 모델 제안

### 2.1 계층 구조

```
Facility (센터/시설)
  └── FacilityCourt[] (구장/면)
        ├── courtName        : "4층 B코트", "실외 2구장"
        ├── floorLevel       : "4층", "지하1층", "1층"
        ├── courtNumber      : "B코트", "2구장"
        ├── floorMaterial    : "우레탄" | "인조잔디" | "마루" | "콘크리트" | ...
        ├── ceilingHeight   : number | null  (m, 실내용)
        ├── officialSpec     : boolean | null (정식 규격 여부, 실외용)
        ├── isExclusiveUse   : boolean        (단독 사용 여부)
        ├── images           : string[]       (해당 구장 전용 현장 사진)
        ├── directionsGuide  : string         (찾아가는 길 텍스트)
        └── indoorOutdoor    : "indoor" | "outdoor"
```

### 2.2 엔티티 관계

| 엔티티 | 설명 |
|--------|------|
| `Facility` | 기존 시설 엔티티. 센터 전체 정보(주소, 전화, 운영시간 등) |
| `FacilityCourt` | 신규. 시설 하위의 개별 구장. `facilityId` FK |

### 2.3 하위 호환성

- **Phase 1**: 기존 시설만 있는 경우 → "전체" 또는 "메인구장" 같은 **가상 구장 1개**로 처리
- **Phase 2**: 구장을 추가한 시설 → 예약/매치가 구장 단위로 연결됨

---

## 3. 시설 관리자(사장님) 등록 UX

### 3.1 공간 계정 구조

```
[시설 상세/수정 화면]
├── 기본정보 (시설명, 주소, 전화, 운영시간, 가격 ...)
├── 구장 목록 (Courts)
│   ├── [+ 구장 추가]
│   └── [구장 1] [구장 2] [구장 3] ...
└── 저장
```

- 시설 1개당 **N개 구장** 등록 가능
- 구장 추가 시: 신규 입력 또는 **기존 정보 복사**

### 3.2 "기존 정보 복사하기" 플로우

```
[구장 추가] 클릭
    → 모달/폼 오픈
    → "빈 상태로 시작" | "기존 구장 정보 복사"

[기존 구장 정보 복사] 선택 시
    → 드롭다운: 복사할 구장 선택 (4층 A코트, 3층 B코트 ...)
    → [복사 후 수정] 클릭
    → 폼에 복사된 값 로드 (층, 바닥재질, 층고, 단독사용 등)
    → 사용자가 **층수/구역명만 변경** (예: A코트 → B코트)
    → 사진·찾아가는 길은 별도 수정
    → 저장
```

**복사되는 항목**
- 층수/구역 명칭 (참고용, 반드시 수정 유도)
- 바닥 재질
- 층고/규격, 정식 규격 여부
- 단독 사용 여부
- 실내/실외

**복사 후 수정 권장**
- 층수/구역 명칭
- 개별 현장 사진
- 찾아가는 길

### 3.3 구장별 입력 항목

| 항목 | 타입 | 필수 | 설명 |
|------|------|------|------|
| 층수/구역 명칭 | text | ○ | "4층 B코트", "실외 2구장" |
| 개별 현장 사진 | image[] | △ | 해당 층/면 전용 사진 |
| 바닥 재질 | select | ○ | 우레탄, 인조잔디, 마루, 콘크리트, 모래, 기타 |
| 층고 (실내) | number(m) | - | 실내일 때만 |
| 정식 규격 여부 (실외) | boolean | - | 실외일 때만 |
| 단독 사용 여부 | boolean | ○ | 옆 코트와 붙어있는지 |
| 찾아가는 길 | textarea | △ | "엘리베이터 2호기", "주차장 뒤 철문" |

## 4. 일반 유저 정보 노출

### 4.1 매치 카드 상세화

**현재**
```
[매치 카드]
○○풋살센터
2/16(월) 18:00
```

**변경**
```
[매치 카드]
○○풋살센터 · 3층 A코트
2/16(월) 18:00 | 우레탄 | 실내
```

- `시설명 + 구장명`을 한 세트로 표시
- 바닥재질, 실내/실외는 배지로 노출 (선택)

### 4.2 필터 확장

기존: 종목, 지역, 실내/실외  
추가: **바닥 재질** (우레탄, 인조잔디, 마루 등)

## 5. 예약 완료 후 "입장 안내" 메시지 레이아웃

### 5.1 목표

- 해당 구장의 **상세 위치**(층, 번호) 강조
- **찾아가는 길**이 눈에 잘 띄도록 구성

### 5.2 레이아웃 제안

```
┌─────────────────────────────────────────┐
│  📍 입장 안내                            │
├─────────────────────────────────────────┤
│                                         │
│  [강조 박스]                             │
│  ○○풋살센터 · 3층 A코트                  │
│  서울시 강남구 ○○로 123                  │
│                                         │
├─────────────────────────────────────────┤
│  📌 찾아오는 길                          │
│  • 엘리베이터 2호기 이용 → 3층 하차       │
│  • 왼쪽 복도 끝 A코트                    │
│  • 주차장 뒤쪽 철문으로도 진입 가능        │
├─────────────────────────────────────────┤
│  📅 2/16(월) 18:00 ~ 20:00              │
│  👥 인원: 4/10명                         │
└─────────────────────────────────────────┘
```

### 5.3 구현 포인트

1. **예약 확정 시점**에 구장 정보(`FacilityCourt`)와 시설 기본 정보를 합쳐서 안내 메시지 생성
2. `directionsGuide`가 있으면 "찾아오는 길" 섹션 표시, 없으면 시설 주소만
3. 알림/문자/이메일 등 채널별로 위 레이아웃에 맞게 포맷팅

---

## 6. 구현 단계 제안

### Phase 1: 데이터 모델 & 관리자 입력
1. `FacilityCourt` 엔티티 추가
2. 시설 수정 화면에 "구장 목록" 섹션 추가
3. 구장별 입력 폼 (층/명칭, 바닥재질, 층고, 단독사용, 사진, 찾아가는 길)
4. **"기존 구장 정보 복사"** 버튼/플로우

### Phase 2: 매치·예약 연동
1. 그룹(매치) ↔ `FacilityCourt` 연결
2. 매치 카드에 "시설명 · 구장명" 표시
3. 예약 시 구장 선택 (해당 시설에 구장이 여러 개일 때)

### Phase 3: 유저 경험
1. 필터에 "바닥 재질" 추가
2. 예약 확정/입장 안내에 구장 상세 + 찾아가는 길 반영
3. 알림/카카오톡 등 전송 포맷 정리

## 7. "기존 정보 복사" UX 상세

### 7.1 시나리오

사장님이 5층에 A·B·C 3개 코트를 같은 규격으로 운영하는 경우:

1. 첫 번째 구장 "5층 A코트" 입력 (바닥: 우레탄, 층고: 4m, 단독: N)
2. 두 번째 구장 추가 시 → **"5층 A코트 정보 복사"** 선택
3. 자동 입력: 바닥 우레탄, 층고 4m, 단독 N
4. **층수/구역 명칭만** "5층 B코트"로 수정
5. B코트 전용 사진 1~2장 등록
6. 찾아가는 길은 A와 동일하다면 그대로, 다르면 수정

### 7.2 UI 플로우

```
[구장 추가] 버튼
    ↓
┌──────────────────────────────────┐
│  새 구장 등록                     │
│  ─────────────────────────────  │
│  ○ 빈 상태로 시작                 │
│  ○ 기존 구장 정보 복사            │
│     └ [5층 A코트 ▼] 선택          │
│     └ [복사 후 수정] 버튼         │
└──────────────────────────────────┘
```

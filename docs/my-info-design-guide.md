# 내 정보(My Info) 페이지 디자인·UX 가이드

Blue & White 테마 일관성을 유지하면서, 성취·계정·푸터 영역의 디자인·UX 개선 방향을 정리한 문서입니다.

---

## 1. 디자인 시스템

### 1.1 성취(Achievement) · 보유 뱃지 섹션

- **적용 완료**
  - **주요 업적**: 그라데이션 배경(`from-amber-50 to-orange-50`), 테두리·그림자(`shadow-lg shadow-amber-500/10`), 달성 카드에 링·그림자로 강조.
  - **보유 뱃지**: 인디고·퍼플 그라데이션 카드(`from-indigo-50 to-purple-50`), 동일한 그림자 톤으로 일관성 유지.
- **추가 제안**
  - 뱃지 호버 시 툴팁(업적 설명·획득일) 노출 시 사용자 이해도 향상.
  - 다크모드 대비: `dark:from-amber-950/30` 등 이미 적용되어 있음.

### 1.2 계정 정보 탭 — 시각적 위계(Visual Hierarchy)

- **적용 완료**
  - **그룹 라벨**: "읽기 전용" / "변경 가능"을 소제목(uppercase, tracking-wider)으로 구분.
  - **읽기 전용**: 배경 `bg-[var(--color-bg-primary)]/50`, 구분선(`divide-y`), "수정 불가"는 작은 뱃지+테두리로 정보성만 표시.
  - **변경 가능**: 비밀번호 등 액션 항목은 별도 블록으로 분리하고, "변경" 버튼은 `font-semibold`·`shadow-sm`으로 강조.
- **가이드**
  - 수정 불가: `text-xs`, `bg-[var(--color-bg-secondary)]`, `border`, 라벨만 사용(버튼 형태 지양).
  - 변경/저장 버튼: `bg-[var(--color-blue-primary)]`, `font-semibold`, 한 행에서 오른쪽 정렬 권장.

## 2. UX 및 기능 추가 제안

### 2.1 선호 활동 시간대 · 주 활동 클럽/장소

- **선호 활동 시간대**
  - **데이터**: 이미 `mercenaryAvailability`(용병 활동 가능 요일·시간)가 있음. 용병 구직자 대시보드에서 편집 가능.
  - **프로필 탭 배치 제안**: **"프로필 정보" 카드 바로 아래** 또는 "용병 기록" 위에 **"매칭에 도움이 되는 정보"** 블록을 두고, 그 안에 다음을 배치.
    1. **선호 활동 시간대**: `mercenaryAvailability`를 요일·시간대 문자열로 표기(예: "월 18:00–21:00, 토 09:00–12:00"). 편집은 "용병 구직자 대시보드"로 이동 링크 또는 인라인 요약 + [편집] 버튼.
    2. **주 활동 클럽/장소**: 신규 필드라면 `preferredClubs` 또는 `mainPlayground` 같은 문자열/배열을 백엔드에 추가 후, 같은 "매칭에 도움이 되는 정보" 블록에 표시·편집.
  - **위치 권장**: "프로필 정보"와 "용병 기록" 사이. 스포츠 파트너 매칭에 중요한 정보이므로 상단에 두는 것이 유리.

### 2.2 매너 점수 시각화

- **적용 완료**
  - `MannerScoreGauge` 컴포넌트: 0–100 원형 게이지, 그린/옐로/레드 구간 색상, 하단에 등급 라벨(그린카드 등).
- **추가 제안**
  - 리스트/카드에서 작은 게이지(`size="sm"`) 사용, 상세 모달에서는 `size="lg"` + "매너가 좋은 선수" 등 짧은 설명 문구 노출.

### 2.3 로그아웃 · 회원 탈퇴 푸터

- **적용 완료**
  - **로그아웃**: 상단 푸터 블록에 단독 배치, `max-w-xs`로 모바일에서 크기 제한해 "일반 액션"으로 인지되도록 함.
  - **회원 탈퇴**: 별도 "위험 영역" 카드(`border-red-200`, `bg-red-50/50`)로 분리, 설명 문구("탈퇴 시 복구할 수 없습니다") + 빨간 톤 버튼으로 실수 클릭 위험 감소.
- **가이드**
  - 두 버튼을 한 줄에 붙이지 말고, 푸터 내에서 위(로그아웃) → 아래(위험 영역) 순서 유지.

## 3. 기술적 구현 — 반응형 레이아웃

### 3.1 계정 정보 긴 리스트 (모바일)

- **현재**
  - `space-y-6 sm:space-y-8`로 그룹 간 간격, `divide-y`로 행 구분, `flex-col sm:flex-row`로 라벨·값·버튼 정렬.
- **추천 구조**
  1. **그룹 단위 스택**: "읽기 전용" / "변경 가능" / "연락처·기타" / "소셜 계정" 등 섹션별로 한 블록씩 세로 쌓기. 모바일에서 스크롤 시 섹션 제목이 보이도록.
  2. **한 행 구조 (모바일)**: 각 항목을 `flex flex-col gap-2`로 두고, 모바일에서는 라벨 → 값 → (수정 불가/변경 버튼) 순으로 세로 배치. `min-h-[44px]` 터치 영역 유지.
  3. **접기(선택)**: "연락처·기타" 등 항목이 많을 경우, `details`/`summary` 또는 아코디언으로 접어두고 펼치면 전체 입력 폼 표시.
  4. **스크롤 앵커**: `id`를 부여해 "계정 정보" 탭 내 앵커 링크(예: #account-readonly)로 이동 가능하게 하면 긴 폼에서 유리.

### 3.2 Tailwind 반응형 패턴

```html
<!-- 그룹 컨테이너 -->
<div class="space-y-6 sm:space-y-8">
  <!-- 읽기 전용 그룹 -->
  <div class="space-y-1">
    <h3 class="text-xs font-semibold uppercase tracking-wider ...">읽기 전용</h3>
    <div class="divide-y ... rounded-xl ... p-2 sm:p-3">
      <!-- 각 행: 모바일 세로, 데스크 가로 -->
      <div class="flex flex-col sm:flex-row sm:items-center justify-between gap-2 py-3 px-2 sm:px-3">
        <div class="flex items-center space-x-3 min-w-0">...</div>
        <span class="shrink-0">수정 불가</span>
      </div>
    </div>
  </div>
</div>
```

- `min-w-0`: flex 자식이 텍스트 잘림 없이 줄어들 수 있게 함.
- `shrink-0`: 버튼·뱃지가 눌리지 않도록 함.
- `pb-24 md:pb-12`: 하단 네비/푸터에 가려지지 않도록 패딩 확보.
